# Анализ PDF-документов

Универсальный пайплайн для PDF: с текстовым слоем, сканов и смешанных. Детерминированная часть (PDF → текст) — в `scripts/extract_pdf.sh`, структурирование и резюме — работа Claude.

## Зависимости (однократно)

```
brew install poppler tesseract tesseract-lang
```

Из poppler используется `pdftotext`, `pdftoppm`, `pdfinfo` — все идут одним пакетом.

## Workflow

### Шаг 1 — Уточнить язык документа

Если язык не указан пользователем и не очевиден из контекста — спросить. Коды Tesseract: `deu`, `eng`, `rus`, `fra`, `spa` и т.д. Можно комбинировать через `+`, например `deu+eng`. Язык нужен, даже если документ с текстовым слоем, — на случай, если отдельные страницы придётся прогонять через OCR (смешанные PDF).

### Шаг 2 — Извлечение текста через скрипт

```
bash .claude/skills/doc-analyze/scripts/extract_pdf.sh <путь_к_pdf> <язык> [--force-ocr]
```

Скрипт сам решает постранично:
- Если у страницы есть нормальный текстовый слой (`pdftotext -layout` выдаёт ≥30 непробельных символов) — берёт его напрямую. Разделитель: `========== SEITE/PAGE N (text) ==========`.
- Иначе (скан / пустая страница / мусор) — рендерит страницу в PNG (300 DPI) и прогоняет через `tesseract -l <язык>`. Разделитель: `========== SEITE/PAGE N (OCR) ==========`.

На выходе: `<stem>_text.txt` рядом с PDF. В stderr — сводка `Pages: X total, Y text, Z OCR`, которая показывает, сколько страниц пришло из какого источника.

**Флаг `--force-ocr`** — принудительно прогнать все страницы через Tesseract, игнорируя текстовый слой. Когда использовать:
- `_text.txt` пришёл из текстового слоя, но содержит много `?`, квадратиков, нечитаемых последовательностей — признак встроенного шрифта без Unicode-маппинга.
- Пользователь явно просит «сделай OCR» / «не доверяй тексту в PDF».
- Есть подозрение, что «текстовый слой» — это на самом деле кривой OCR из другого софта.

**Важно:** Tesseract в sandbox не видит `/tmp` — скрипт сознательно кладёт промежуточные PNG рядом с PDF и чистит за собой через `trap cleanup EXIT`.

### Шаг 3 — Создать `<stem>_text.md`

Структурированный markdown на базе `<stem>_text.txt`:
- разделы по страницам или смысловым секциям документа
- таблицы для табличных данных (ключ — значение, параметры)
- не переводить текст, но можно добавлять пояснения на русском там, где это помогает чтению
- сохранять оригинальные цифры/коды/VIN/номера без изменений

### Шаг 4 — Создать `<stem>_resume.md`

Резюме для пользователя на русском:
- что это за документ и зачем он нужен
- ключевые данные и факты
- что важно или подозрительно
- что сделать (если применимо — например, «сохранить оригинал» или «проверить у дилера»)

### Шаг 5 — Вывести в чат

Краткое резюме (4–8 предложений) с главными выводами и списком созданных файлов (пути относительно корня проекта).

## Правила для артефактов

- Все три файла (`_text.txt`, `_text.md`, `_resume.md`) сохраняются **в той же папке, где лежит PDF**
- Имя файлов — от основы (stem) исходного PDF; пробелы и спецсимволы в именах допустимы (скрипт их обрабатывает)
- Не редактировать `_text.txt` вручную — это сырой извлечённый текст, нужен для верификации и повторного анализа
- В `_text.md` можно править структуру, но не заменять цитируемые куски

## Ограничения извлечения

**Для страниц с маркером `(OCR)`** Tesseract может ошибаться на:
- юридических терминах с редкими словами
- цифрах, похожих на буквы (`0`/`O`, `1`/`l`, `8`/`B`)
- рукописных пометках
- таблицах со сложной вёрсткой

**Для страниц с маркером `(text)`** `pdftotext -layout` обычно даёт точный результат, но:
- порядок колонок в сложных вёрстках может ломаться
- встроенные шрифты без корректного Unicode-маппинга дают мусор на месте букв — в этом случае нужен `--force-ocr`

Если в `_text.txt` видишь подозрительное (например, нетипичная сумма в контракте или странный VIN) — упомяни это в резюме и предложи пользователю визуально перепроверить. Посмотри на маркер `(text)` / `(OCR)` возле номера страницы: это подсказывает, какой источник использовался, и насколько ему можно доверять.

## Язык

Пояснения и резюме — на русском. Цитаты и ключевые термины из документа — в оригинале (немецкий, английский и т.д.).
