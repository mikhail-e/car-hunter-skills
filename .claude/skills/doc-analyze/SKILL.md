# OCR-анализ документов

Пайплайн для работы со сканированными PDF (фото страниц без текстового слоя). Детерминированная часть (PDF→PNG→TXT) — в `scripts/ocr_pdf.sh`, структурирование и резюме — работа Claude.

## Зависимости (однократно)

```
brew install poppler tesseract tesseract-lang
```

## Workflow

### Шаг 1 — Уточнить язык документа

Если язык не указан пользователем и не очевиден из контекста — спросить. Коды Tesseract: `deu`, `eng`, `rus`, `fra`, `spa` и т.д. Можно комбинировать через `+`, например `deu+eng`.

### Шаг 2 — OCR через скрипт

```
bash .claude/skills/doc-analyze/scripts/ocr_pdf.sh <путь_к_pdf> <язык>
```

Скрипт сам:
- делает `pdftoppm -png -r 300` во временную папку рядом с PDF
- прогоняет `tesseract -l <язык>` по каждой странице
- склеивает постранично в `<stem>_text.txt` с разделителями `========== SEITE/PAGE N ==========`
- удаляет временные PNG
- выводит путь к созданному файлу

**Важно:** Tesseract в sandbox не видит `/tmp` — скрипт сознательно кладёт PNG рядом с PDF и чистит за собой.

### Шаг 3 — Создать `<stem>_text.md`

Структурированный markdown на базе `<stem>_text.txt`:
- разделы по страницам или смысловым секциям документа
- таблицы для табличных данных (ключ — значение, параметры)
- не переводить текст, но можно добавлять пояснения на русском там, где это помогает чтению
- сохранять оригинальные цифры/коды/VIN/номера без изменений

### Шаг 4 — Создать `<stem>_resume.md`

Резюме для пользователя на русском:
- что это за документ и зачем он нужен
- ключевые данные и факты
- что важно или подозрительно
- что сделать (если применимо — например, «сохранить оригинал» или «проверить у дилера»)

### Шаг 5 — Вывести в чат

Краткое резюме (4–8 предложений) с главными выводами и списком созданных файлов (пути относительно корня проекта).

## Правила для OCR-артефактов

- Все три файла (`_text.txt`, `_text.md`, `_resume.md`) сохраняются **в той же папке, где лежит PDF**
- Имя файлов — от основы (stem) исходного PDF; пробелы и спецсимволы в именах допустимы (скрипт их обрабатывает)
- Не редактировать `_text.txt` вручную — это сырой OCR, нужен для верификации и повторного анализа
- В `_text.md` можно править структуру, но не заменять цитируемые куски

## OCR не идеален

Tesseract ошибается на:
- юридических терминах с редкими словами
- цифрах, похожих на буквы (`0`/`O`, `1`/`l`, `8`/`B`)
- рукописных пометках
- таблицах со сложной вёрсткой

Если в `_text.txt` видишь что-то подозрительное (например, нетипичная сумма в контракте или странный VIN) — упомяни это в резюме и предложи пользователю визуально перепроверить.

## Язык

Пояснения и резюме — на русском. Цитаты и ключевые термины из документа — в оригинале (немецкий, английский и т.д.).
