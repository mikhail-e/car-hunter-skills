# Поиск и оценка новых EV объявлений

Автоматический поиск новых объявлений на mobile.de по сохранённому URL, оценка каждого по критериям ev-eval, сохранение результатов.

## Workflow

### Шаг 0 — Сброс Playwright

1. Удалить все файлы в `.playwright-mcp/` и `temp/`: `python3 scripts/cleanup.py`
2. Закрыть все предыдущие сессии браузера через `browser_close`, чтобы избежать ошибок «Target page, context or browser has been closed». Ошибку при закрытии игнорировать (может не быть активной сессии).

### Шаг 1 — Загрузить конфиг

Прочитать файл `search-config.json` из корня проекта:
- `searchUrl` — URL страницы поиска на mobile.de
- `checkedIds` — массив ID уже проверенных объявлений

### Шаг 1.5 — Закрыть cookie-баннер

После первого `browser_navigate` на mobile.de выполнить **один раз**:

```
browser_run_code:
  code: |
    async (page) => {
      try {
        // Попытка 1: кнопка прямо на странице (основной вариант mobile.de)
        const btn = await page.waitForSelector('button:has-text("Einverstanden")', { timeout: 5000 });
        if (btn) await btn.click();
      } catch(e1) {
        try {
          // Попытка 2: внутри iframe (запасной вариант)
          const frame = await page.waitForSelector('iframe[id*="sp_message_iframe"]', { timeout: 3000 });
          if (frame) {
            const f = await frame.contentFrame();
            const btn2 = await f.waitForSelector('button:has-text("Einverstanden")', { timeout: 3000 });
            if (btn2) await btn2.click();
          }
        } catch(e2) { /* баннер не найден — ОК */ }
      }
      await page.waitForTimeout(1000);
    }
```

Этот шаг выполняется **только один раз** — при первом открытии mobile.de в сессии. Повторять при открытии каждого объявления не нужно.

### Шаг 2 — Открыть страницу поиска

1. Открыть `searchUrl` через **Playwright MCP** (`browser_navigate`)
2. Закрыть cookie-баннер (Шаг 1.5) — только если это первая навигация в сессии
3. Сделать snapshot страницы (`browser_snapshot`)
3. Извлечь ID объявлений: `python3 scripts/extract_ids.py temp/search-page.md checkedId1,checkedId2,...`
   Скрипт выведет новые (не проверенные) ID по одному на строку и итоговую строку `TOTAL:X NEW:Y`.
4. Если TOTAL = 0 — сообщить пользователю и **завершить**. Это нормальный результат, не нужно менять фильтры или предлагать расширить поиск.

### Шаг 3 — Фильтрация

Фильтрация уже выполнена скриптом `extract_ids.py` (передаются checkedIds). Если NEW = 0 — сообщить пользователю и завершить.

### Шаг 4 — Оценка каждого нового объявления

Для каждого нового объявления выполнить полный анализ по критериям из скилла ev-eval:

1. Открыть страницу объявления через Playwright (`browser_navigate`)
2. Сделать snapshot (`browser_snapshot`)
3. Извлечь все характеристики: цена, пробег, год, мотор, батарея, владельцы, происхождение, дилер, рейтинг, комплектация
4. Проверить 11 dealbreakers (см. критерии ниже)
5. Проверить nice-to-have опции
6. Сформировать вердикт: ✅ / ⚠️ / ❌

### Шаг 5 — Сохранить результаты

После оценки каждого объявления:

1. **Обновить `evaluations.md`**:
   - Определить следующий порядковый номер (максимальный существующий + 1)
   - Добавить строку в сводную таблицу **сверху** (сразу после заголовка таблицы)
   - Добавить полную карточку оценки **сверху** (сразу после сводной таблицы, перед предыдущими карточками)

2. **Обновить `search-config.json`**:
   - Добавить ID объявления в массив `checkedIds`

### Шаг 6 — Итоговая сводка

Вывести в чат:
- Всего объявлений на странице: X
- Новых (не проверенных): Y
- Результаты: Z ✅ / W ⚠️ / V ❌

## Критерии оценки

### Dealbreakers (если хоть один явно провален → ❌)

| # | Критерий | Порог |
|---|----------|-------|
| 1 | Количество мест | = 5 |
| 2 | Батарея (netto) | ≥ 77 кВтч |
| 3 | Адаптивный круиз | ACC stop&go |
| 4 | Контроль мёртвых зон | Side Assist |
| 5 | Задний парктроник | Да |
| 6 | Без аварий (unfallfrei) | Да |
| 7 | Рейтинг дилера | ≥ 4,0 И рекомендации ≥ 70% |
| 8 | Количество владельцев | ≤ 2 |
| 9 | Происхождение | Deutsche Ausführung |
| 10 | Зоны климат-контроля | ≥ 2 |
| 11 | Тепловой насос | Да |

### Nice to have

- Мотор ≥ 150 кВт (204 л.с.)
- SoH батареи
- Батарейный сертификат (AVILOO и т.п.)
- GW-гарантия
- Подогрев сидений
- Подогрев руля
- Стояночный обогрев
- AR проекционный дисплей (HUD)
- Travel Assist
- Камера 360°
- Камера заднего вида
- LED фары
- 2 комплекта колёс
- Электропривод сидений

### Вердикты

- **✅ Подходит** — все dealbreakers пройдены → звоним, договариваемся на осмотр
- **⚠️ Уточнить** — dealbreakers не провалены, но 1–2 не подтверждены → звоним, уточняем
- **❌ Не подходит** — хотя бы один dealbreaker явно провален → не тратим время

## Формат карточки в evaluations.md

Каждая карточка содержит 4 блока (идентично ev-eval):

### Блок 1 — Карточка машины

| Параметр | Значение |
|----------|----------|
| Дилер | название, город, языки |
| Рейтинг дилера | ⭐ X.X / рекомендации Y% |
| Расстояние до Мюнхена | ~X км |
| Цена | €XX XXX |
| Пробег | XX XXX км |
| Регистрация | MM/YYYY |
| Мотор | XXX кВт (XXX л.с.) |
| Владельцев | X |
| Ссылка | URL |

### Блок 2 — Dealbreakers

| # | Критерий | Порог | Факт | Статус |
|---|----------|-------|------|--------|
| 1 | ... | ... | ... | ✅/❌/❓ |

### Блок 3 — Nice to have

| Опция | Статус |
|-------|--------|
| ... | ✅/❌/❓ |

### Блок 4 — Резюме и вердикт

2–3 предложения + **Вердикт: ✅/⚠️/❌**

## Формат сводной таблицы

```
| # | Модель | Цена | Пробег | Год | Вердикт | Дата проверки | Ссылка |
```

## Работа с Playwright

- Python-логика вынесена в `scripts/` — вызывать как `python3 scripts/<name>.py`
- Snapshot файлы сохранять в `temp/` (например `temp/search-page.md`, `temp/listing-<ID>.md`). Папка `temp/` в `.gitignore`.
- При работе со snapshot: искать ключевые слова (Kilometerstand, Leistung, Erstzulassung, Fahrzeughalter, Klimatisierung, Einparkhilfe, Wärmepumpe, Totwinkel, Abstandstempomat, unfallfrei, Sitzheizung, Head-Up, и т.д.)
