# Поиск и оценка новых EV объявлений

Автоматический поиск новых объявлений на mobile.de по сохранённому URL, оценка каждого по критериям ev-eval, сохранение результатов.

## Workflow

### Шаг 1 — Сброс Playwright

1. Удалить все файлы в `.playwright-mcp/` и `temp/`: `python3 scripts/cleanup.py`
2. Закрыть все предыдущие сессии браузера через `browser_close`, чтобы избежать ошибок «Target page, context or browser has been closed». Ошибку при закрытии игнорировать (может не быть активной сессии).

### Шаг 2 — Загрузить конфиг

Прочитать файл `search-config.json` из корня проекта:
- `searchUrl` — URL страницы поиска на mobile.de
- `checkedIds` — массив ID уже проверенных объявлений

### Шаг 3 — Открыть страницу поиска

1. Открыть `searchUrl` через **Playwright MCP** (`browser_navigate`)
2. Закрыть cookie-баннер (только при первом открытии в сессии): `browser_run_code` с `filename: scripts/dismiss_cookie.js`
3. Сделать snapshot страницы (`browser_snapshot`), сохранить в `temp/search-page.md`
4. Извлечь ID объявлений: `python3 scripts/extract_ids.py temp/search-page.md checkedId1,checkedId2,...`
   Скрипт извлекает ID **только из реальных результатов поиска** (`ref=srp` в URL), игнорируя рекомендации («Ähnliche Fahrzeuge», `ref=srx`) и рекламу.
   Выведет новые (не проверенные) ID по одному на строку и итоговую строку `TOTAL:X NEW:Y`.
5. Если TOTAL = 0 — сообщить пользователю и **завершить**. Это нормальный результат, не нужно менять фильтры или предлагать расширить поиск.
6. Если NEW = 0 — сообщить пользователю и завершить.

### Шаг 4 — Оценка и сохранение каждого нового объявления

Для каждого нового объявления **последовательно** выполнить оценку и сразу сохранить результат, прежде чем переходить к следующему:

1. Открыть страницу объявления через Playwright (`browser_navigate`)
2. Сделать snapshot (`browser_snapshot`), сохранить в `temp/listing-<ID>.md`
3. Извлечь все характеристики и сформировать карточку оценки **по критериям из скилла ev-eval** (dealbreakers, nice-to-have, вердикт)
4. **Сохранить результат**:
   - Обновить `search-config.json`: добавить ID объявления в массив `checkedIds` (всегда, независимо от вердикта)
   - Если вердикт **✅ или ⚠️** — обновить `evaluations.md`: определить следующий порядковый номер (максимальный существующий + 1), добавить строку в сводную таблицу **сверху** (сразу после заголовка таблицы), добавить полную карточку оценки **сверху** (сразу после сводной таблицы, перед предыдущими карточками)
   - Если вердикт **❌** — в `evaluations.md` не добавлять
5. Перейти к следующему объявлению

### Шаг 5 — Подготовка сообщений дилерам

После оценки всех новых объявлений — для каждого с вердиктом **✅ или ⚠️** подготовить сообщение дилеру по шаблону из `templates/dealer-message.md`:

1. Прочитать шаблон `templates/dealer-message.md`
2. Для каждого подходящего объявления:
   - Заполнить вводную часть: название дилера, модель, номер объявления, вопрос про Unfallfrei
   - Собрать уточняющие вопросы из ❓ в dealbreakers и nice-to-have (SoH, сертификат батареи и т.д.)
   - Добавить блок про резервирование и осмотр
   - Добавить просьбу о переходе на английский
3. Вывести готовое сообщение в чат
4. Поставить ✉️ в колонке «Письмо» в сводной таблице `evaluations.md`

### Шаг 6 — Закрыть браузер

Закрыть браузер через `browser_close`.

### Шаг 7 — Итоговая сводка

Вывести в чат:
- Всего объявлений на странице: X
- Новых (не проверенных): Y
- Результаты: Z ✅ / W ⚠️ / V ❌

## Формат карточки в evaluations.md

Идентичен формату вывода из скилла ev-eval (4 блока: карточка машины, dealbreakers, nice-to-have, резюме и вердикт).

## Формат сводной таблицы

```
| # | Модель | Цена | Пробег | Год | Вердикт | Письмо | Дата проверки | Ссылка |
```

- Колонка «Письмо»: ✉️ если сообщение дилеру подготовлено, пусто если нет

## Работа с Playwright

- Python-логика вынесена в `scripts/` — вызывать как `python3 scripts/<name>.py`
- Snapshot файлы сохранять в `temp/`. Папка `temp/` в `.gitignore`.
- При работе со snapshot: искать ключевые слова (Kilometerstand, Leistung, Erstzulassung, Fahrzeughalter, Klimatisierung, Einparkhilfe, Wärmepumpe, Totwinkel, Abstandstempomat, unfallfrei, Sitzheizung, Head-Up, и т.д.)
